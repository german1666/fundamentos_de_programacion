Documentación — Diagrama ER (Biblioteca Escolar)
1. Introducción

El presente documento describe el Diagrama Entidad-Relación (ER) diseñado para un sistema de gestión de una biblioteca escolar. El modelo fue creado en dbdiagram.io y representa las entidades principales, atributos y relaciones necesarias para manejar libros, autores, estudiantes y préstamos.

2. Entidades y atributos

Autor

id_autor (PK)

nombre

nacionalidad

Libro

id_libro (PK)

titulo

isbn (único)

anio_publicacion

id_autor (FK → Autor.id_autor)

Estudiante

id_estudiante (PK)

nombre

grado

Préstamo

id_prestamo (PK)

id_libro (FK → Libro.id_libro)

id_estudiante (FK → Estudiante.id_estudiante)

fecha_prestamo

fecha_devolucion (opcional, null = préstamo activo)

3. Relaciones y cardinalidades

Autor 1 — N Libro: Un autor puede escribir muchos libros.

Libro 1 — N Préstamo: Un libro puede registrarse en varios préstamos (históricos).

Estudiante 1 — N Préstamo: Un estudiante puede realizar múltiples préstamos.

4. Reglas de negocio

Unicidad de ISBN: Cada libro debe tener un ISBN único.

Un préstamo activo por libro: Solo puede existir un préstamo sin fecha de devolución por libro.

Integridad referencial: No se deben eliminar autores, libros o estudiantes que tengan préstamos relacionados.
